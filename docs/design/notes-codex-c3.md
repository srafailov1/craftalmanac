# Codex C3 — Flush Thresholds

`data/flush-thresholds.json` is generated by
`scripts/build_flush_thresholds.mjs`.

The generator reads `foodSpeciesCatalog` from `app.js` and includes only
species with `category: "mushroom"`. At the time of this deliverable the
whitelist is:

- `morel`

The morel threshold is `25 mm` rain in `72 hours`, marked:

- `confidence: "low"`
- `ownerReview: true`

Rationale: the sources support morel fruiting as a cool, moist spring
phenomenon, with rain/moisture mattering, but they do not provide a defensible
national `mm/72h` trigger. The value is therefore a conservative UI gate for
owner review, not a biological promise.

Verification:

- `node --check scripts/build_flush_thresholds.mjs`
- `node scripts/build_flush_thresholds.mjs`
- `node scripts/build_flush_thresholds.mjs --verify`

The verifier checks that the output exactly matches the current edible
mushroom whitelist, every threshold is positive, each window is 72 hours, and
low-confidence owner-review status remains attached.
