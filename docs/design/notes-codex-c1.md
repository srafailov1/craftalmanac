# Codex C1 — Tide Station Index

`data/tide-stations.json` is generated by `scripts/build_tide_stations.mjs`
from the NOAA CO-OPS metadata API:

- `https://api.tidesandcurrents.noaa.gov/mdapi/prod/webapi/stations.json?type=tidepredictions`

Each station record keeps only:

```json
{"id":"8454000","name":"Providence, State Pier no.1","lat":41.80717,"lng":-71.40067}
```

The index includes NOAA tide-prediction stations whose metadata carries a US
state or territory code, including Alaska and Hawaii. Providence `8454000` is
required and verified by the script.

Integration helper spec:

```js
function findNearestTideStation(stations, lat, lng) {
  // Inputs: station array from data/tide-stations.json and a forecast
  // location in decimal degrees.
  // Output: the nearest station object plus distanceKm.
  // Method: Haversine distance over the in-memory station list. This list is
  // small enough for a direct scan whenever the forecast location changes.
}
```

Suggested UI cache key: round forecast lat/lng to 3 decimals before looking up
and fetching CO-OPS tide predictions, so small map movements do not churn
station or tide requests.

Verification:

- `node --check scripts/build_tide_stations.mjs`
- `node scripts/build_tide_stations.mjs`
- `node scripts/build_tide_stations.mjs --verify`

The `--verify` check compares generated records to individual CO-OPS metadata
responses for Providence RI, San Francisco CA, Key West FL, Honolulu HI, and
Anchorage AK.
