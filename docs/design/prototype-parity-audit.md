## data-points — 8 (high 3, medium 5, low 0)

### [HIGH] Access ring is drawn INSIDE the fill instead of as an outer ring
- **aspect:** layout
- **prototype:** occ-core fill radius 5.5 with circle-stroke-width 2.4 painted OUTSIDE the fill edge — the access-status color forms an outer ring around the species core (prototype/index.html:746-747)
- **relaunch:** Canvas icon: fill circle radius 6.5, then drawMarkerOutline strokes a circle at radius 5.3 (smaller than the 6.5 fill) so the access outline sits INSIDE/atop the fill, not around it (app.js:3503-3512)
- **gap:** Owner sees the access-status color as a thin ring carved inside the colored dot rather than a distinct outer collar around it; the dot reads as one fill with a faint inner edge instead of a core+ring.
- **fix:** In app.js ensureMarkerIcon (lines 3503-3512), make the access ring exterior, mirroring Mapbox circle-stroke semantics (stroke painted outward from the fill edge = centered at fillRadius + lineWidth/2). Keep the fill drawn first, then stroke an outline whose radius exceeds the fill radius. Concrete change: set fillRadius=5.5 and outlineRadius = 5.5 + (1.8/2) = 6.4 (or adopt prototype width: lineWidth 2.4, outlineRadius 6.7), i.e. before -> after: `const fillRadius = 6.5; const outlineRadius = 5.3;` -> `const fillRadius = 5.5; const outlineRadius = fillRadius + lineWidth / 2;` (pass lineWidth so it stays in sync; 1.8 -> optionally 2.4 to match prototype stroke width). The 26px canvas (MARKER_ICON_SIZE=26, center 13, app.js:488) easily fits radius ~6.7. Leave the drawMarkerOutline-after-fill ordering as-is — that part is already correct; only the radius relationship is wrong.
