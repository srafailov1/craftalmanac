# Codex C2 — Phenology Histograms

`data/phenology/food.json`, `data/phenology/ink.json`, and
`data/phenology/medicine.json` are generated by
`scripts/build_phenology_histograms.mjs` from the iNaturalist histogram
endpoint:

- `https://api.inaturalist.org/v1/observations/histogram`

The generator reads the production species catalogs from `app.js`, then writes
one 12-value relative-abundance array per species id. Each array is normalized
to that species' highest monthly count, so values are comparable within a
species curve but not as cross-species population estimates.

Filter choices:

- Food plants use the iNaturalist "Flowers and Fruits: Fruits or Seeds"
  annotation (`term_id=12`, `term_value_id=14`), so fruit/nut/berry curves
  track harvest-relevant observations.
- Food fungi use the general observation histogram because plant phenology
  annotations do not apply.
- Ink species use "Fruits or Seeds" except honeysuckle and goldenrod, which
  use "Flowers" (`term_id=12`, `term_value_id=13`) because the catalog material
  is floral.
- Medicine species use the general observation histogram because the catalog's
  plant parts are mixed and should not be forced into one annotation.

Verification:

- `node --check scripts/build_phenology_histograms.mjs`
- `node scripts/build_phenology_histograms.mjs`
- `node scripts/build_phenology_histograms.mjs --verify`

The verifier checks that every catalog species has a length-12 array, every
array sums above zero, values are finite 0-1 numbers, elderberry peaks in July
through September, and black walnut peaks in September through November.
