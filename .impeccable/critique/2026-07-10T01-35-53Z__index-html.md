## Anti-Patterns Verdict

**Does this look AI-generated? No.** The identity is earned and consistently executed: specimen-label mono voice, Fraunces reserved for identity moments, a live seasonal register system (day→dusk observed switching during the session), and data-as-decoration (a labeled, per-month histogram; ring-vs-fill marker grammar). The two pattern-matching tells — colored card spines and mono uppercase kickers ("THE SHELF · 45 PROFILES") — are named, systematic identity choices used at consistent anchor points, not per-section scaffolding.

**Deterministic scan (CLI):** clean — zero findings across index.html and app.js.

**Deterministic scan (in-page, dusk register, 375px):** six finding families, deduped:
- **low-contrast** — #bcafa5 on #544c5a = 3.8:1 (needs 4.5:1) on `#seasonHistHead`, `.legend-note`, `#dataStatus`. This is the dusk `--reg-sub` token — a documented owner-approved exception (audit_contrast.mjs, 2026-06-13 "prototype look" decision). The detector can't know that; the report below treats only its collision with the error/status channel as actionable.
- **tiny-text** — 10–11px body-classified text across `.season-caveat`, `.legend-note`, `.legend-title`, `#dataStatus`, `.off-hint`, `.off-storage`. Partly the specimen-label identity (labels), but `#dataStatus` and the offline hints are genuine reading copy.
- **wide-tracking** — 0.12em on the same mono runs; identity, false positive as "body text."
- **gpt-thin-border-wide-shadow** — 1px border + 24px blur on every `.floating` panel + welcome modal. Judged mostly false positive: the shell needs separation over an unpredictable map surface, and the shadow is low-opacity; noted because it is also the saturated 2024–25 default combo.
- **cream-palette** — body #f8f6ef. False positive in context: it's the day-register paper of a four-register almanac system, and the body is almost never visible behind the map.
- **layout-transition** — `transition: max-height` on `#seasonHist` and `.legend-body` accordions. Real, minor perf note.

**Visual overlays:** injection succeeded; overlays rendered in the Browser pane highlighting the flagged elements (cream palette banner on body, hairline-border+shadow and layout-animation tags on the season bar, low-contrast/tiny-text tags on the status line). Live overlay server was started and stopped cleanly.

**Where the two assessments agree/disagree:** the human pass and detector agree on the status-line typography being the weak point (tiny + low contrast in dusk). The detector caught the layout-property transitions the human pass missed. The human pass caught everything the detector structurally cannot: the deep-link boot gap, the search-status clobber, touch-target sizes, the illegible off-state chip, and the unactionable caution line.

## Overall Impression

This is a confident, characterful field tool that knows exactly what it is. The point card is the whole product thesis in one component — species, access status, the actual rule, safety, place, source — and "occurrence is never permission" is structurally present at every decision point, not just in a disclaimer. The weak edge is the smallest text in the interface: the status/error channel and micro-labels are where identity (10px mono, tracked, muted) collides with field legibility, and two error paths report the wrong thing. The single biggest opportunity: make the gold caution actionable — it commands "CHECK WHO MANAGES THIS LAND" and then offers no way to do it.

## What's Working

1. **The point card.** ACCESS/RULES/SAFETY/PLACE in exactly four labeled chunks, the gold occurrence-is-not-permission line, focus moved into the card on open, Escape wired, real buttons. It's beautiful, honest, and accessible at once.
2. **The register system delivers the "living palette" promise.** Day→dusk switched live during review; permission-status colors stayed distinguishable; `color-scheme` flips native controls so the date input doesn't flash light-in-dark.
3. **Honest-data voice as a design system.** Record counts with named sources, the cluster-tint caption, offline storage estimates ("~0.1 MB + ~22 MB shared catalog"), "Overview circles count all recorded occurrences… zoom in to map points." Trust is built sentence by sentence.
4. **Accessibility depth beyond the usual.** Per-month text alternatives on histogram bars, correct combobox pattern, 20 focus-visible rules, live region deliberately placed outside an innerHTML-rebuilt panel.

## Priority Issues

- **[P1] Deep-link boot into the point band can render an empty map.** Booting at `#map=food&c=…&z=14.5`: both chunk manifests fetched, zero chunk fetches, no status message, blank map — while the zoom-in path correctly fetched 50 chunks. Likely mechanism: the boot-time load runs against the pre-jump view (below z8 → returns empty) and the hash `jumpTo` never re-triggers it. **Verify on an allowlisted origin** (403 confound). Why it matters: shared views are the teacher workflow ("bookmark the pipestone view" is in the code comments). Fix: after the boot view applies, kick the chunk loader on first idle. → `/impeccable harden`
- **[P2] Search network failure reads as "No matching places found."** Reproduced: app.js:5349 sets "Search unavailable. Try again in a moment." then :5364 unconditionally overwrites it because the failed fetch returned `[]`. In the field this tells a user their place doesn't exist when the truth is they're offline. Fix: only set the no-matches message when the fetch succeeded. → `/impeccable harden`
- **[P2] Touch targets below floor on mobile.** Legend chips 155×24, season-bar text buttons ~24px, popup close 12×27 (Escape exists, but not on touch). WCAG 2.2 minimum is 24×24 — several controls sit at or under it; a field tool warrants the 44px comfort floor. Fix: hit-area padding, not visual redesign. → `/impeccable adapt`
- **[P2] The smallest text carries the most important messages.** `#dataStatus` — the channel for "The map can't load right now" — renders 11px, tracked, in dusk's approved-below-floor #bcafa5 (3.8:1). The off-state legend chip dims to 0.32 opacity, making "Prohibited" illegible exactly where a user could learn the map hides that data by default. Fix without touching the approved sub hue: let status text switch to `--reg-ink` when `kind: "error"`, bump status/offline-hint copy to ≥12px, and dim only the off-chip's swatch while keeping its label readable. → `/impeccable polish` (or `typeset` for the size scale)
- **[P2, opportunity] The caution commands an action the UI doesn't offer.** "OCCURRENCE IS NOT PERMISSION, CHECK WHO MANAGES THIS LAND" has no affordance — no manager name even where the PAD-US polygon under the point knows it. Naming the manager ("This is NY State Parks land") would convert the product's core warning from a scold into a service. → `/impeccable shape`

## Persona Red Flags

**Casey (distracted mobile field user):** 24px chips and a 12px-wide close button, one-handed, at dusk, possibly gloved. Conditions rail clips mid-item at the right viewport edge with no scroll cue. The dusk register at actual dusk, though, is exactly right for this persona.

**Sam (screen reader / keyboard):** Strongest persona showing. Histogram months announce their stacked counts; suggestions use aria-activedescendant; offline announcements survive panel rebuilds. Misses: nav buttons announce a stray "·" ("MAPS·"), and the off-state chip communicates state only via opacity.

**Jordan (first-timer):** Three near-synonyms for uncertainty on one card — "Private / unchecked", "Access not sourced", "Access unknown in Falling Fruit" — plus a legend chip called "Unverified". Each is technically distinct (status/why/provenance) but a newcomer reads the same fact three times and wonders what they're missing.

## Minor Observations

- Sheet-to-sheet switching (Materials → Projects) appeared to need two taps: the first closes the open sheet, the second opens the new one. Medium confidence (synthetic clicks) — worth a hand check.
- Radar weather overlay is on by default and visually loud at the national view (pink/blue cells compete with cluster circles). There is a toggle, but the first impression spends attention the data earned.
- `transition: max-height` on the season chart and legend accordions animates layout; small panels so cost is low, but `grid-template-rows` is a free upgrade.
- Project-card titles clip mid-word at row edges (the scroll-peek pattern earns it; the clipped Fraunces looks slightly untidy).
- Day slider is desktop-only; mobile gets "Set date" — a reasonable simplification, verified that "Back to now" appears once a date is set.
- Night register not visually exercised this run (tokens read consistent); printable field-card pages not reviewed.

## Questions to Consider

1. Prohibited-off is a deliberate default (documented in code) — should the map whisper it once, e.g. "12 prohibited sites hidden" in the status line, so the withholding itself is legible rather than discoverable only via a dimmed chip?
2. Does live radar earn default-on at the national view, or is weather a conditions-rail concern until asked for — with radar auto-on only at field zooms where it informs "should I go today"?
3. If the popup can name the land manager, does the RULES row become the place where the almanac teaches the *next verb* — "call, check the compendium, ask at the visitor center" — per manager type?
4. The identity's smallest voice (10px tracked mono) is doing double duty as both flavor and infrastructure. Which runs are truly labels (keep), and which are messages wearing a label costume (promote)?
